# IGP 基础（内部网关协议）

> [English](igp.md) · **简体中文**

本文介绍内部网关协议（IGP）的基本概念。IGP 与 [BGP](bgp.md) 相对：BGP 负责自治系统*之间*的路由，IGP 负责单个自治系统*内部*的路由。建议先阅读 [BGP 基础](bgp.md)。

---

## IGP 与 EGP 的区别

路由协议按作用范围分为两类：

| 类别 | 作用范围 | 代表协议 |
|---|---|---|
| EGP（外部网关协议） | 自治系统*之间* | BGP |
| IGP（内部网关协议） | 单个自治系统*内部* | OSPF、IS-IS、RIP 等 |

IGP 使一个自治系统内部的路由器了解如何到达本系统内的各个网段；BGP 则在自治系统之间交换前缀。两者分工不同，通常同时运行。

## 常见的 IGP

| 协议 | 类型 | 说明 |
|---|---|---|
| OSPF | 链路状态 | 应用广泛；支持 IPv4/IPv6（OSPFv3）；按区域（area）划分以利扩展 |
| IS-IS | 链路状态 | 常见于大型运营商骨干；协议无关、易扩展 |
| RIP | 距离矢量 | 早期协议；配置简单，但收敛慢、规模受限 |
| EIGRP | 距离矢量（高级） | 常见于 Cisco 环境 |

## 链路状态与距离矢量

IGP 主要有两种工作方式：

- **链路状态**（OSPF、IS-IS）：每台路由器都掌握全网拓扑，各自使用最短路径算法（如 Dijkstra）独立计算到各目的地的路径。收敛快、扩展性好。
- **距离矢量**（RIP）：路由器只把「到某网段需要几跳」告知直接相连的邻居，信息逐跳传播。实现简单，但收敛较慢，且需额外机制防止环路。

## IGP 与 BGP 如何协作

在同时运行 IGP 与 BGP 的自治系统中，二者各司其职：

- **IGP** 提供系统内部的可达性，包括到达 BGP 邻居与路由*下一跳*的内部路径。
- **BGP** 负责自治系统之间的前缀，并依赖 IGP 解析这些路由的下一跳实际如何到达。

简而言之：IGP 决定域内如何路由，BGP 决定域间存在哪些前缀。

## 本网络的内部路由

在本网络中，各 PoP 通过 **IPv6 骨干网络**（`2001:db8::/44`）互联，内部路由使用 **iBGP**（内部 BGP）分发，而非传统 IGP。

在这种由隧道构成、规模有限的全连接骨干上，iBGP（配合直连或静态路由解析下一跳）已足以满足内部可达性，因此未额外运行 OSPF 或 IS-IS。对外前缀仍通过 eBGP 向上游与对等方宣告（见 [BGP 基础](bgp.md)）。
