# BGP 基础
> [English](bgp.md) · **简体中文**

本文面向初次接触域间路由的读者，介绍边界网关协议（BGP）的基本概念。建议先阅读 [自治系统（AS）与 ASN 基础](asn.md)。

---

## 什么是 BGP

BGP（Border Gateway Protocol，边界网关协议）是互联网的域间路由协议，负责在不同的[自治系统（AS）](asn.md)之间交换可达性信息，决定数据如何跨越多个网络抵达目的地。

## BGP 解决的问题

每个自治系统只直接了解自身内部的网络。BGP 使相邻的自治系统相互告知各自可达的 IP 前缀及其路径。这些信息在全网逐跳传递，最终在任意两个端点之间建立可达性。

## 前缀与路由通告

自治系统通过 BGP 通告（announce）其所负责的 IP 前缀（例如 `2001:db8:50::/44`）。收到通告的其他自治系统据此得知应将发往这些地址的流量转发给谁。

一条 BGP 路由的核心信息是目的前缀及到达该前缀的路径。

## AS_PATH

每条 BGP 路由都带有一个 AS_PATH，记录其依次经过的 ASN 序列。它有两个主要作用：

- 选路：在多条可达路径中，AS_PATH 较短者通常更优。
- 防环：自治系统若在 AS_PATH 中看到自身的 ASN，则丢弃该路由，以避免路由环路。

位于路径末端、最初通告该前缀的自治系统称为 origin AS。

## eBGP 与 iBGP

| 类型 | 用途 |
|---|---|
| eBGP（外部 BGP） | 在不同自治系统之间交换路由 |
| iBGP（内部 BGP） | 在同一自治系统内部的路由器之间同步路由 |

## 选路概述

BGP 选路不仅依据跳数，而是按固定优先级依次评估一系列属性进行决策，常见的包括 Local Preference、AS_PATH 长度、MED 等。因此 BGP 的结果是策略驱动的，未必是物理上的最短路径。

## 互联方式：transit 与 peering

| 方式 | 说明 |
|---|---|
| Transit（中转） | 向上游运营商付费的关系，以换取到达整个互联网的可达性 |
| Peering（对等互联） | 两个网络直接交换各自及其客户的路由，通常互不付费，多在 IXP 进行 |

## 相关概念

- IXP（互联网交换中心）：多个网络在同一物理设施集中互联的场所。
- 全表（full table）：一个自治系统从其连接收到的全部互联网路由条目。
- 路由来源校验：见下节。

## 安全：路由来源校验

BGP 协议本身不验证通告的真伪，因此存在前缀误通告与路由劫持的风险。RPKI 通过签名的 ROA 声明哪个自治系统有权通告哪个前缀。路由器据此对收到的路由执行来源校验（ROV），将其判定为 valid、invalid 或 unknown。网络发布自身的 ROA，并对互联中收到的路由执行路由来源校验（参见对 [互联](peering.md) 的要求）。
